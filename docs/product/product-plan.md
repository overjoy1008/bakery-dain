# Product Plan

## Purpose

Bakery Dain은 인스타그램 DM 중심의 수동 예약을 웹 기반 프리오더 흐름으로 정리하는 프로젝트입니다. 고객은 메뉴와 픽업 가능일을 한눈에 보고 예약하고, 운영자는 만들 수 있는 날짜와 메뉴만 열어 주문을 통제합니다.

## Positioning

```txt
작은 홈베이커리
예약 제작
픽업 중심
한정 수량
선물하기 좋은 구움과자와 계절 디저트
```

Sprout Artisan Bakery처럼 프리오더 사이트의 구조와 신뢰감을 가져오되, 다닷네 베이커리의 귀여운 프로필 이미지, 손에 들고 찍은 제품 사진, 시즌 이벤트 이미지, 소량 제작 감성을 더합니다.

## Primary Personas

### Customer

- 인스타그램에서 다닷네를 보고 주문하려는 사람
- 선물용 구움과자나 디저트를 찾는 사람
- 픽업 가능한 날짜와 메뉴를 빠르게 확인하고 싶은 사람
- DM보다 명확한 주문 확인을 원하는 사람

### Admin

- 하루 몇 시간씩 직접 굽는 1인 또는 소규모 운영자
- 만들 수 있는 날과 없는 날이 매주 달라질 수 있음
- 날짜별로 받을 메뉴가 달라질 수 있음
- 복잡한 쇼핑몰 관리자보다 빠른 예약 관리가 필요함

## Product Principles

- 날짜 우선: 고객은 먼저 픽업일을 고르고, 그 날짜에 가능한 메뉴를 봅니다.
- 소량 제작: 무한 재고처럼 보이지 않게 남은 수량과 마감 상태를 명확히 보여줍니다.
- 운영자 보호: 주문 마감, 휴무, 품절, 픽업 시간 제한을 관리자가 쉽게 설정합니다.
- 신뢰감: 사진, 가격, 픽업 안내, 예약 완료 확인이 일관되게 제공됩니다.
- 작은 브랜드 감성: 문구는 대형 쇼핑몰보다 친근하고 담백하게 씁니다.

## Customer MVP Flow

1. 메인에서 이번 주 메뉴와 예약 안내를 확인합니다.
2. 예약 가능일 캘린더에서 픽업 날짜를 선택합니다.
3. 해당 날짜에 주문 가능한 메뉴를 확인합니다.
4. 메뉴와 수량을 선택합니다.
5. 이름, 연락처, 요청사항을 입력합니다.
6. 예약 내용을 확인하고 제출합니다.
7. 예약 완료 화면에서 픽업일, 메뉴, 수량, 금액, 안내사항을 확인합니다.

## Admin MVP Flow

1. 관리자가 로그인합니다.
2. 이번 주 또는 특정 기간의 날짜 상태를 설정합니다.
3. 날짜별 픽업 시간과 주문 마감 시간을 설정합니다.
4. 날짜별 주문 가능 메뉴와 수량을 설정합니다.
5. 예약 목록을 확인합니다.
6. 예약 상태를 접수, 확인, 준비중, 픽업완료, 취소로 변경합니다.

## Content Model

- 브랜드 소개
- 메인 슬라이드 이미지
- 움직이는 안내 문구
- 메뉴 카테고리
- 메뉴 카드
- 예약 가능일
- 픽업 안내
- 환불/변경 안내
- 예약 완료 안내

## Success Definition

첫 운영 버전은 고객 문의를 줄이고, 주문 누락을 줄이고, 운영자가 그 주에 받을 수 있는 주문량을 통제할 수 있으면 성공입니다.
