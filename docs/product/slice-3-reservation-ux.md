# Slice 3. Reservation UX Shell

## Goal

메뉴를 보고 예약 주문으로 이어지는 고객 흐름을 API 없이 먼저 검증합니다. 이 단계는 최종 예약 로직이 아니라, 메뉴 선택, 픽업일 선택, 회원/비회원 정보 입력, 영수증 확인의 사용감을 점검하는 프론트엔드 MVP입니다.

## Pages

- `/menu`: 메뉴를 자세히 보고 예약 페이지로 진입하는 페이지
- `/reserve`: 메뉴, 캘린더, 예약자 정보, 영수증 위젯을 한 흐름으로 확인하는 페이지

## UX Direction

- 고객은 메뉴를 먼저 보고 예약 주문을 시작합니다.
- 픽업 가능일은 메뉴 선택 이후 해당 메뉴를 받을 수 있는 날짜만 고를 수 있게 보여줍니다.
- 비회원 예약은 이름과 연락처만으로 빠르게 진행합니다.
- 회원 예약은 회원 번호 또는 인스타 DM 닉네임을 함께 남길 수 있게 합니다.
- 영수증은 선택 메뉴, 수량, 픽업일, 픽업 시간, 남은 수량, 합계를 즉시 반영합니다.

## TereneHandover Reference

- Calendar: 날짜별 상태 맵을 UI에 반영하는 구조를 참고합니다.
- Receipt: 선택값을 기준으로 금액과 예약 요약을 계속 갱신하는 구조를 참고합니다.
- ReservationForm: 예약자 정보와 동의 체크를 제출 조건으로 묶는 구조를 참고합니다.

Bakery Dain에서는 숙박 예약의 체크인/체크아웃, 쿠폰, 마일리지, 결제 복잡도는 제외하고, 홈베이커리 예약 주문에 필요한 최소 흐름만 남깁니다.

## Current Limits

- 모든 데이터는 mock입니다.
- 예약 완료는 실제 저장이 아니라 UI 상태 변경입니다.
- 실제 수량 차감, 마감 검증, 예약 번호 생성은 Slice 7에서 API로 연결합니다.
